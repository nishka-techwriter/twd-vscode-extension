# Класс HtmlTagFoldingProvider

## Общее описание

`HtmlTagFoldingProvider` - это класс, реализующий интерфейс `vscode.FoldingRangeProvider`. Он предоставляет функциональность для сворачивания (фолдинга) участков кода в текстовом редакторе VS Code на основе пользовательских HTML-тегов. Класс автоматически определяет области между тегами `<folding-start>` и `<folding-end>` и делает их сворачиваемыми.

Основные особенности:
- Реализует интерфейс `vscode.FoldingRangeProvider` для работы с системой фолдинга VS Code.
- Использует стеки для корректной обработки вложенных областей.
- Поддерживает пользовательские теги для разметки сворачиваемых блоков.
- Работает автоматически при открытии документов поддерживаемых типов.
- Не требует ручного вызова команд - интегрируется напрямую в редактор.

## Архитектура класса

### Структура класса

#### HtmlTagFoldingProvider
- **Назначение:** Предоставляет области сворачивания для пользовательских HTML-тегов
- **Реализует:** `vscode.FoldingRangeProvider` с методом `provideFoldingRanges()`
- **Механизм работы:** Поиск пар тегов `<folding-start>` и `<folding-end>` в документе
- **Особенность:** Использует стек для обработки вложенных областей

## Подробный разбор класса

### Класс HtmlTagFoldingProvider

```typescript
export class HtmlTagFoldingProvider implements vscode.FoldingRangeProvider {
  provideFoldingRanges(
    document: vscode.TextDocument
  ): vscode.ProviderResult<vscode.FoldingRange[]> {
    const ranges: vscode.FoldingRange[] = [];
    const stack: number[] = [];

    for (let line = 0; line < document.lineCount; line++) {
      const text = document.lineAt(line).text;

      if (text.includes('<folding-start>')) {
        stack.push(line);
      }

      if (text.includes('<folding-end>') && stack.length > 0) {
        const start = stack.pop()!;
        if (line > start) {
          ranges.push(
            new vscode.FoldingRange(start, line, vscode.FoldingRangeKind.Region)
          );
        }
      }
    }

    return ranges;
  }
}
```

#### Подробное объяснение работы класса

`export class HtmlTagFoldingProvider implements vscode.FoldingRangeProvider`
- Объявление класса, который реализует интерфейс `vscode.FoldingRangeProvider`
- Интерфейс требует реализации метода `provideFoldingRanges()`
- Этот интерфейс является частью системы фолдинга (сворачивания кода) VS Code

`provideFoldingRanges(document: vscode.TextDocument): vscode.ProviderResult<vscode.FoldingRange[]>`
- Основной метод, который вызывается VS Code для получения областей сворачивания
- Принимает один параметр: `document` - текущий текстовый документ
- Возвращает `vscode.ProviderResult<vscode.FoldingRange[]>` - массив областей для сворачивания
- `ProviderResult` - специальный тип VS Code, который может быть массивом, промисом или `null`

`const ranges: vscode.FoldingRange[] = [];`
- Создает пустой массив для хранения найденных областей сворачивания
- `vscode.FoldingRange[]` - тип TypeScript для массива объектов `FoldingRange`
- Каждая область будет представлена объектом `vscode.FoldingRange`

`const stack: number[] = [];`
- Создает стек для отслеживания начала областей
- `number[]` - массив чисел, где будут храниться номера строк с тегами `<folding-start>`
- Стек необходим для корректной обработки вложенных областей

`for (let line = 0; line < document.lineCount; line++) {`
- Начинает цикл по всем строкам документа
- `document.lineCount` - общее количество строк в документе
- `line` - текущий индекс строки (начинается с 0)

`const text = document.lineAt(line).text;`
- Получает текст текущей строки
- `document.lineAt(line)` - возвращает объект строки по индексу
- `.text` - свойство объекта строки, содержащее текст строки

`if (text.includes('<folding-start>')) {`
- Проверяет, содержит ли текущая строка тег начала области
- `includes()` - метод строки, который проверяет наличие подстроки
- Если тег найден, выполняется код в блоке `if`

`stack.push(line);`
- Добавляет номер текущей строки в стек
- `push()` - метод массива, который добавляет элемент в конец
- Номер строки сохраняется для последующего создания области сворачивания

`if (text.includes('<folding-end>') && stack.length > 0) {`
- Проверяет два условия:
  1. Содержит ли строка тег конца области (`<folding-end>`)
  2. Не пуст ли стек (есть ли соответствующий открывающий тег)
- Оба условия должны быть истинными для создания области

`const start = stack.pop()!;`
- Извлекает номер строки начала области из стека
- `pop()` - метод массива, который удаляет и возвращает последний элемент
- `!` - оператор утверждения ненулевого значения (TypeScript)
- Гарантирует TypeScript, что `pop()` не вернет `undefined` (т.к. проверили `stack.length > 0`)

`if (line > start) {`
- Проверяет, что строка конца находится после строки начала
- Защита от некорректной разметки (например, `<folding-end>` перед `<folding-start>`)

`ranges.push(new vscode.FoldingRange(start, line, vscode.FoldingRangeKind.Region));`
- Создает новую область сворачивания и добавляет ее в массив `ranges`
- `new vscode.FoldingRange(start, line, vscode.FoldingRangeKind.Region)`:
  - `start` - номер строки начала области (где `<folding-start>`)
  - `line` - номер строки конца области (где `<folding-end>`)
  - `vscode.FoldingRangeKind.Region` - тип области (регион)
- `vscode.FoldingRangeKind.Region` - специальный маркер, который VS Code использует для стилизации области

`return ranges;`
- Возвращает массив найденных областей сворачивания
- Если массив пустой, VS Code не покажет никаких областей для сворачивания
- Если массив содержит элементы, VS Code отобразит значки сворачивания рядом с соответствующими строками

**Что делает:** Автоматически находит все пары тегов `<folding-start>` и `<folding-end>` в документе и делает области между ними сворачиваемыми.

**Когда использовать:** В больших документах, где нужно скрывать подробные разделы, примеры кода или дополнительную информацию для улучшения читаемости.

## Интеграция в расширение

### 1. Регистрация в `extension.ts`

Провайдер регистрируется в главном файле расширения:

```typescript
// Общий селектор для провайдеров
const selector = [
    { language: 'markdown', scheme: 'file' },
    { language: 'mdx', scheme: 'file' },
    { language: 'html', scheme: 'file' }
];

// Провайдер для <folding-start> / <folding-end>
context.subscriptions.push(
    vscode.languages.registerFoldingRangeProvider(selector, new HtmlTagFoldingProvider())
);
```

#### Подробное объяснение регистрации

`const selector = [ ... ];`
- Создает массив селекторов, определяющих типы документов, где работает провайдер
- Селекторы указывают, что провайдер активен для:
  - Markdown файлов (`.md`)
  - MDX файлов (`.mdx`)
  - HTML файлов (`.html`)
- `scheme: 'file'` означает, что работают только с локальными файлами

`context.subscriptions.push( ... )`
- Добавляет провайдер в подписки контекста расширения
- Обеспечивает правильное освобождение ресурсов при деактивации расширения

`vscode.languages.registerFoldingRangeProvider(selector, new HtmlTagFoldingProvider())`
- Регистрирует `HtmlTagFoldingProvider` как провайдер областей сворачивания
- `vscode.languages.registerFoldingRangeProvider()` - метод VS Code API для регистрации провайдеров фолдинга
- Первый параметр - селектор (определяет, где работает провайдер)
- Второй параметр - экземпляр провайдера (`new HtmlTagFoldingProvider()`)

**Автоматическая активация:**

## Пример использования

### Сценарий 1: Сворачивание подробных примеров кода

```
# Документация API

<folding-start>

:::warning

Некоторый текст

:::

<folding-end>

```


## Краткое описание
Основные моменты использования API...
```

**Что происходит:**
1. При открытии файла `HtmlTagFoldingProvider` анализирует документ
2. Находит теги `<folding-start>` и `<folding-end>`
3. Создает область сворачивания от строки с `<folding-start>` до строки с `<folding-end>`
4. В редакторе появляется стрелочка для сворачивания раздела с подробным примером

### Сценарий 2: Вложенные области сворачивания
```markdown
<folding-start>
# Глава 1

<folding-start>
## Раздел 1.1

<folding-start>
### Подраздел 1.1.1
Содержимое подраздела...
<folding-end>

<folding-end>

<folding-end>
```

**Что происходит:**
1. Провайдер использует стек для обработки вложенных тегов
2. Первый `<folding-end>` закрывает самый внутренний блок
3. Второй `<folding-end>` закрывает блок уровня 1.1
4. Третий `<folding-end>` закрывает блок Главы 1
5. Создаются три независимые области сворачивания с правильной вложенностью

### Сценарий 3: Ошибочная разметка
```markdown
<folding-end> <!-- Ошибка: закрывающий тег без открывающего -->

<folding-start>
Правильная область
<folding-end>
```

**Что происходит:**
1. При обнаружении `<folding-end>` проверяется `stack.length > 0`
2. Для первого тега стек пустой, поэтому область не создается
3. Для второй пары тегов все корректно, область создается
4. Провайдер устойчив к ошибкам разметки

# Класс InfoBlocksFoldingProvider

## Общее описание

`InfoBlocksFoldingProvider` - это класс, реализующий интерфейс `vscode.FoldingRangeProvider`. Он предоставляет функциональность для сворачивания (фолдинга) участков кода в MDX-документах на основе пользовательских маркеров `:::`. Класс автоматически определяет области между открывающим тегом (например, `:::note`) и закрывающим тегом (`:::`) и делает их сворачиваемыми.

Основные особенности:
- Реализует интерфейс `vscode.FoldingRangeProvider` для работы с системой фолдинга VS Code.
- Использует стеки для корректной обработки вложенных областей.
- Поддерживает MDX-директивы с идентификаторами и атрибутами.
- Работает автоматически при открытии MDX-документов.
- Не требует ручного вызова команд - интегрируется напрямую в редактор.

---

## Архитектура класса

### Структура класса

#### InfoBlocksFoldingProvider
- **Назначение:** Предоставляет области сворачивания для MDX-блоков с маркерами `:::`
- **Реализует:** `vscode.FoldingRangeProvider` с методом `provideFoldingRanges()`
- **Механизм работы:** Поиск пар открывающих тегов (например, `:::note`) и закрывающих тегов (`:::`)
- **Особенность:** Использует регулярные выражения для точного определения границ блоков и стек для обработки вложенных областей

---

## Подробный разбор класса

### Класс InfoBlocksFoldingProvider

```typescript
export class InfoBlocksFoldingProvider implements vscode.FoldingRangeProvider {
  provideFoldingRanges(document: vscode.TextDocument): vscode.ProviderResult<vscode.FoldingRange[]> {
    const ranges: vscode.FoldingRange[] = [];
    const stack: number[] = [];

    // Регулярное выражение для открывающего тега:
    // - необязательные пробелы в начале строки (\s*)
    // - три двоеточия (:::)
    // - необязательные пробелы (\s*)
    // - как минимум один непробельный символ (\S) — наличие идентификатора/атрибутов
    const startRegex = /^\s*:::\s*\S/;

    // Регулярное выражение для закрывающего тега:
    // - необязательные пробелы в начале строки
    // - ровно три двоеточия
    // - только пробелы (или конец строки) — больше ничего
    const endRegex = /^\s*:::\s*$/;

    for (let line = 0; line < document.lineCount; line++) {
      const text = document.lineAt(line).text;

      if (startRegex.test(text)) {
        stack.push(line);
      }

      if (endRegex.test(text) && stack.length > 0) {
        const startLine = stack.pop()!;
        if (line > startLine) {
          ranges.push(
            new vscode.FoldingRange(
              startLine,
              line,
              vscode.FoldingRangeKind.Region
            )
          );
        }
      }
    }

    return ranges;
  }
}
```

`export class InfoBlocksFoldingProvider implements vscode.FoldingRangeProvider`
- Объявление класса, который реализует интерфейс `vscode.FoldingRangeProvider`
- Интерфейс требует реализации метода `provideFoldingRanges()`
- Этот интерфейс является частью системы фолдинга (сворачивания кода) VS Code

`provideFoldingRanges(document: vscode.TextDocument): vscode.ProviderResult<vscode.FoldingRange[]>`
- Основной метод, который вызывается VS Code для получения областей сворачивания
- Принимает один параметр: `document` - текущий текстовый документ
- Возвращает `vscode.ProviderResult<vscode.FoldingRange[]>` - массив областей для сворачивания
- `ProviderResult` - специальный тип VS Code, который может быть массивом, промисом или `null`

`const ranges: vscode.FoldingRange[] = [];`
- Создает пустой массив для хранения найденных областей сворачивания
- `vscode.FoldingRange[]` - тип TypeScript для массива объектов `FoldingRange`
- Каждая область будет представлена объектом `vscode.FoldingRange`

`const stack: number[] = [];`
- Создает стек для отслеживания начала областей
- `number[]` - массив чисел, где будут храниться номера строк с открывающими тегами `:::name`
- Стек необходим для корректной обработки вложенных областей

`const startRegex = /^\s*:::\s*\S/;`
- Регулярное выражение для поиска открывающего тега
- `^` - начало строки
- `\s*` - ноль или более пробельных символов
- `:::` - точное совпадение трех двоеточий
- `\s*` - ноль или более пробельных символов (позволяет писать `::: note` или `:::   note`)
- `\S` - один непробельный символ - гарантирует наличие идентификатора или атрибута
- Без этого символа строка `:::` была бы ошибочно распознана как открывающий тег

`const endRegex = /^\s*:::\s*$/;`
- Регулярное выражение для поиска закрывающего тега
- `^` - начало строки
- `\s*` - ноль или более пробельных символов
- `:::` - точное совпадение трех двоеточий
- `\s*` - ноль или более пробельных символов
- `$` - конец строки
- Требует, чтобы в строке **не было ничего, кроме** `:::` и пробелов

`for (let line = 0; line < document.lineCount; line++) {`
- Начинает цикл по всем строкам документа
- `document.lineCount` - общее количество строк в документе
- `line` - текущий индекс строки (начинается с 0)

`const text = document.lineAt(line).text;`

- `.text` - свойство объекта строки, содержащее текст строки

`if (startRegex.test(text)) {`
- Проверяет, соответствует ли текущая строка шаблону открывающего тега
- `test()` - метод регулярного выражения, возвращает `true` при совпадении
- Если строка начинается с `:::`, после которого (возможно, через пробелы) есть непробельный символ - это открывающий тег
- Примеры совпадений: `:::note`, `::: tip`, `:::{.note}`, `:::warning{title="Важно"}`
- Примеры НЕ-совпадений: `:::`, `:::   `, `Обычный текст :::note`

`stack.push(line);`

`if (endRegex.test(text) && stack.length > 0) {`
- Проверяет два условия:
  1. Соответствует ли строка шаблону закрывающего тега (только `:::` и пробелы)
  2. Не пуст ли стек (есть ли соответствующий открывающий тег)
- Оба условия должны быть истинными для создания области

`const startLine = stack.pop()!;`
- Извлекает номер строки начала области из стека
- `pop()` - метод массива, который удаляет и возвращает последний элемент
- `!` - оператор утверждения ненулевого значения (TypeScript)
- Гарантирует TypeScript, что `pop()` не вернет `undefined` (т.к. проверили `stack.length > 0`)

`if (line > startLine) {`
- Проверяет, что строка конца находится после строки начала
- Защита от некорректной разметки (например, закрывающий тег перед открывающим)

`ranges.push(new vscode.FoldingRange(startLine, line, vscode.FoldingRangeKind.Region));`
- Создает новую область сворачивания и добавляет ее в массив `ranges`
- `new vscode.FoldingRange(startLine, line, vscode.FoldingRangeKind.Region)`:
  - `startLine` - номер строки начала области (где открывающий тег, например `:::note`)
  - `line` - номер строки конца области (где закрывающий тег `:::`)
  - `vscode.FoldingRangeKind.Region` - тип области (регион)
- `vscode.FoldingRangeKind.Region` - специальный маркер, который VS Code использует для стилизации области

`return ranges;`
- Возвращает массив найденных областей сворачивания
- Если массив пустой, VS Code не покажет никаких областей для сворачивания
- Если массив содержит элементы, VS Code отобразит значки сворачивания рядом с соответствующими строками

**Что делает:** Автоматически находит все пары MDX-директив (открывающий тег вида `:::name` и закрывающий тег `:::`) в документе и делает области между ними сворачиваемыми.

**Когда использовать:** В MDX-документах с заметками, предупреждениями, врезками, кастомными компонентами и другими блоками, оформленными через синтаксис `:::`.

## Интеграция в расширение

### 1. Регистрация в `extension.ts`

Провайдер регистрируется в главном файле расширения:

```typescript
// Селектор для MDX-файлов
const selector = [
    { language: 'mdx', scheme: 'file' }
];

// Провайдер для MDX-блоков :::note / :::
context.subscriptions.push(
    vscode.languages.registerFoldingRangeProvider(selector, new InfoBlocksFoldingProvider())
);
```

#### Подробное объяснение регистрации

`const selector = [ ... ];`
- Создает массив селекторов, определяющих типы документов, где работает провайдер
- Селектор указывает, что провайдер активен для MDX-файлов (`.mdx`)
- `scheme: 'file'` означает, что работают только с локальными файлами

`context.subscriptions.push( ... )`
- Добавляет провайдер в подписки контекста расширения
- Обеспечивает правильное освобождение ресурсов при деактивации расширения

`vscode.languages.registerFoldingRangeProvider(selector, new InfoBlocksFoldingProvider())`
- Регистрирует `InfoBlocksFoldingProvider` как провайдер областей сворачивания
- `vscode.languages.registerFoldingRangeProvider()` - метод VS Code API для регистрации провайдеров фолдинга
- Первый параметр - селектор (определяет, где работает провайдер)
- Второй параметр - экземпляр провайдера (`new InfoBlocksFoldingProvider()`)

### 2. Особенности работы провайдера

**Автоматическая активация:**
- Провайдер активируется автоматически при открытии MDX-файлов
- Не требует вызова команд или настройки со стороны пользователя
- Работает в фоновом режиме, анализируя документ при каждом изменении

**Отображение в интерфейсе VS Code:**
- Области сворачивания отображаются как стрелочки на полях редактора
- При нажатии на стрелочку область сворачивается/разворачивается
- Свернутые области показываются как `...` с возможностью развернуть

### 3. Активация в `package.json`

Для автоматической активации провайдера в `package.json` используются события активации:

```json
"activationEvents": [
    "onLanguage:mdx"
]
```

**Объяснение:**
- `onLanguage:mdx` - расширение активируется при открытии MDX-файлов
- Это событие обеспечивает загрузку провайдера только когда он нужен

## Пример использования

### Сценарий 1: Базовый блок с именем

```mdx
:::note
Это важная заметка, которую можно свернуть.
Содержимое может занимать несколько строк.
:::
```

**Что происходит:**
1. При открытии файла `InfoBlocksFoldingProvider` анализирует документ
2. Находит открывающий тег `:::note` (строка 1)
3. Находит закрывающий тег `:::` (строка 4)
4. Создает область сворачивания от строки 1 до строки 4
5. В редакторе появляется стрелочка для сворачивания блока заметки

### Сценарий 2: Блок с атрибутами

```mdx
:::warning{title="Внимание" icon="⚠️"}
Это предупреждение с заголовком и иконкой.
Пользователь может свернуть его, чтобы не отвлекаться.
:::
```

**Что происходит:**
1. Регулярное выражение `startRegex` находит `:::warning{title="Внимание"...}`
2. Символ `{` после пробелов является непробельным символом (`\S`) - тег распознается как открывающий
3. Провайдер корректно обрабатывает директиву с атрибутами

### Сценарий 3: Вложенные блоки

```mdx
:::parent
Родительский блок

:::child
Дочерний блок, который можно свернуть отдельно
:::

Продолжение родительского блока
:::
```

**Что происходит:**
1. Стек: `[0]` (parent на строке 0)
2. Найден открывающий тег `:::child` на строке 2 → стек: `[0, 2]`
3. Найден закрывающий тег `:::` на строке 4 → извлекается `2`, создается область (2-4) для дочернего блока
4. Найден закрывающий тег `:::` на строке 6 → извлекается `0`, создается область (0-6) для родительского блока
5. В редакторе появляются две независимые стрелочки: одна сворачивает дочерний блок, другая - родительский

### Сценарий 4: Блок без имени, но с классом

```mdx
:::{.note}
Специфичный синтаксис MDX-директив
:::
```

**Что происходит:**
1. Регулярное выражение `startRegex` находит `:::{.note}`
2. Символ `{` после `:::` является непробельным символом - тег распознается как открывающий
3. Провайдер корректно создает область сворачивания

### Сценарий 5: Ошибочная разметка

```mdx
::: 
Этот блок не будет свернут, потому что открывающий тег некорректен
:::

<folding-start>
Этот блок обрабатывается HtmlTagFoldingProvider, а не InfoBlocksFoldingProvider
<folding-end>
```

**Что происходит:**
1. Строка `:::` с пробелами после - это закрывающий тег, а не открывающий
2. При обнаружении `:::` на первой строке проверяется `startRegex` - совпадения нет
3. При обнаружении `:::` на четвертой строке проверяется `endRegex` - совпадение есть, но стек пуст
4. Область не создается, провайдер устойчив к ошибкам разметки
5. HTML-теги игнорируются этим провайдером и обрабатываются отдельным `HtmlTagFoldingProvider`
